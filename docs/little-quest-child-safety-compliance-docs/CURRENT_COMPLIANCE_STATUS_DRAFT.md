# Little Quest — 현재까지 기획상 준수 반영 현황 초안

> 기준: 현재까지 대화에서 확정된 Little Quest 기획 내용을 바탕으로 한 1차 자체 점검이다. 실제 저장소 코드 기준 점검은 `CODEX_CHILD_SAFETY_COMPLIANCE_AUDIT_PROMPT.md`를 사용해 별도 수행한다.

## 1. 현재까지 기획에 반영된 항목

| 영역 | 상태 | 메모 |
|---|---|---|
| 사진 원본/AI 전송본 분리 | 반영됨 | 원본 별도 보관, 압축본만 AI API 전송 방향 확정 |
| 목데이터 금지 | 반영됨 | 실제 데이터 기반 테스트 원칙 확정 |
| 위치 지도 전략 | 반영됨 | 국내 NAVER Maps, 해외 Google Maps |
| 위치 권한 거부 대체 | 부분 반영 | 위치 없이 카드 생성 가능 정책을 명확히 문서화 필요 |
| AI 검증 | 반영됨 | Wikipedia/Wikidata/GBIF/iNaturalist/공공 API 기반 검증 방향 확정 |
| 홀로 등급 | 반영됨 | Normal/Holo/Seasonal Holo MVP 기준 |
| 지도 중복 핀 정책 | 반영됨 | 동일 canonical은 최초 위치만 대표 핀, 추가 핀은 슬롯/카드 소모 |
| 개인정보 수정 전 재인증 | 반영됨 | 비밀번호/SNS/선택형 재인증 플로우 정리 |
| 계정 확인 목업 | 반영됨 | 비밀번호 가입자/SNS 가입자/혼합 사용자 화면 설계 |
| 광고 배제 | 방향 반영 | MVP 광고 SDK 미사용을 명확히 문서화 필요 |
| 아이 직접 결제 제한 | 방향 반영 | 부모 모드/부모 게이트 설계 추가 필요 |

## 2. 아직 기획·문서에 추가해야 하는 항목

| 영역 | 필요 작업 |
|---|---|
| 만 14세 미만 보호자 동의 | 회원가입 플로우에 생년월일 기반 분기와 보호자 동의 화면 추가 |
| 보호자 모드 | 아이 데이터/위치/결제/동의 관리 화면 추가 |
| 위치정보 동의 | 위치정보 이용 동의와 위치 권한 요청 시점 정의 |
| 사진 민감정보 감지 | 얼굴/사람/번호판/주소 감지 시 경고 또는 차단 정책 추가 |
| 데이터 삭제 | 계정/사진/카드/위치/동의 철회 API와 UX 추가 |
| 아이용 쉬운 개인정보 안내 | 보호자용 문서와 별도로 쉬운 안내 화면 추가 |
| 스토어 심사 | App Store/Google Play 개인정보 라벨, SDK 목록, 권한 설명 문서 추가 |
| 법무 검토 | 개인정보/위치정보/아동/결제/환불 범위 검토 필요 |

## 3. MVP 출시 전 차단 이슈

아래 항목은 구현 전까지 출시 차단으로 본다.

1. 만 14세 미만 보호자 동의 플로우 부재
2. 개인정보·사진·AI·위치 동의 분리 부재
3. 위치정보 처리 및 삭제 정책 부재
4. 계정/사진/위치/카드 삭제 기능 부재
5. 민감정보 수정 전 재인증 서버 검증 부재
6. 아이 화면 직접 결제 유도 방지 미정의
7. AI 결과 고지 및 위험 행동 금지 프롬프트 미정의
8. 원본/AI 전송본 분리 구현 미확인
